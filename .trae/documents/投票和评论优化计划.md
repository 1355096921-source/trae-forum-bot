# 投票和评论优化计划

## 一、需求概述

用户需要对现有自动化评论脚本进行两项优化：

1. **评论内容优化**：每条评论固定以"已投票"开头，整体控制在 50 字以内
2. **自动投票功能**：在评论前先对帖子进行投票

**当前阶段**：先创建测试脚本，只对指定帖子 `https://forum.trae.cn/t/topic/22549` 执行操作，验证效果后再全量启用。

---

## 二、修改步骤

### 步骤 1：创建测试脚本 test_single_topic.py

新增独立测试脚本，只对帖子 ID 22549 执行：
- 获取帖子详情
- 检查投票状态并投票
- 生成评论（以"已投票"开头，50字以内）
- 发表评论
- **不记录到 state.json**，方便反复测试

### 步骤 2：修改 comment_generator.py

更新 `USER_PROMPT_TEMPLATE`：
- 以"已投票"开头
- 总字数 ≤ 50 字
- 修改 `max_tokens` 为 100

### 步骤 3：修改 discourse_client.py

添加投票方法：
- `vote_post(post_id: int)` — 对帖子投票

## 四、测试流程

1. 用户运行 `python test_single_topic.py`
2. 脚本对帖子 22549 执行投票和评论
3. 用户到论坛查看效果
4. 确认无误后，再修改 `main.py` 全量启用

---

## 五、风险处理

- 投票 API 不匹配：捕获异常并跳过投票，继续评论
- 帖子无投票功能：自动检测并跳过投票
- Cookie 失效：提示用户重新获取
