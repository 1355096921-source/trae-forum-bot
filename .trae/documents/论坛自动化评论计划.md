# 论坛自动化评论任务计划

## 一、任务概述

从 **TRAE 官方中文社区**（Discourse 论坛）的指定分类下自动抓取帖子内容，调用 **DeepSeek API (deepseek-v4-flash)** 生成智能评论，并通过用户提供的 **Cookie/Session** 自动发表评论。通过 **本地 JSON 文件** 记录已评帖子 ID，实现去重，避免重复评论。

目标分类URL: `https://forum.trae.cn/c/38-category/40-category/40/l/new`

## 四、各模块详细设计

### 4.1 `requirements.txt`

依赖包：
- `requests` — HTTP 请求
- `python-dotenv` — 环境变量读取

### 4.2 `.env`（用户手动创建并填写）

```ini
# DeepSeek API 配置
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-v4-flash

# Discourse 论坛登录 Cookie（从浏览器开发者工具复制）
DISCOURSE_COOKIE=_t=your_cookie_here; _forum_session=your_session_here

# 目标分类 URL
TARGET_CATEGORY_URL=https://forum.trae.cn/c/38-category/40-category/40/l/new

# 每次执行最多评论多少条（防止刷太多）
MAX_COMMENTS_PER_RUN=5
```

### 4.3 `config.py`

- 使用 `python-dotenv` 加载 `.env`
- 提供配置项读取和基础校验（如 API Key 是否为空）
- 将 Cookie 字符串解析为 `requests` 可用的字典格式

### 4.4 `discourse_client.py`

封装与 Discourse 论坛的交互：

| 方法 | 功能 | API 端点 |
|------|------|----------|
| `get_latest_topics(category_slug, category_id)` | 获取分类下的最新主题列表 | `GET /c/{slug}/{id}/l/latest.json` |
| `get_topic_detail(topic_id)` | 获取主题详情及帖子正文 | `GET /t/{topic_id}.json` |
| `create_reply(topic_id, raw_content)` | 发表评论/回复 | `POST /posts` |

**技术细节**：
- 所有请求携带 `Cookie` Header 模拟登录态
- `User-Agent` 设置为常见浏览器，降低被封风险
- 对 JSON 响应做异常处理（如 403 表示 Cookie 失效）

### 4.5 `deepseek_client.py`

封装 DeepSeek API 调用：

- 统一调用 `chat.completions` 接口
- 支持可配置的 `temperature`、`max_tokens`
- 异常处理：网络超时、API 限流、余额不足等

### 4.6 `comment_generator.py`

评论生成逻辑：

1. **组装 Prompt**：
   - 系统角色："你是一位热心、专业的社区用户，擅长对技术/创意类帖子给出简短、真诚、有建设性的中文评论。"
   - 用户输入：帖子标题 + 帖子正文（截取前 2000 字符避免过长）
   - 要求："请根据以上内容生成一条 50~150 字的中文评论，要求真诚、具体、有互动感，不要泛泛而谈。"

2. **调用 DeepSeek**：通过 `deepseek_client.py` 获取评论内容

3. **内容清洗**：去除引号、多余空格、AI 痕迹词（如"作为 AI..."）

### 4.7 `state_manager.py`

本地 JSON 状态管理：

- **文件路径**: `state.json`
- **数据结构**:
  ```json
  {
    "commented_topics": [
      {"topic_id": 12345, "commented_at": "2026-07-08T10:30:00"},
      {"topic_id": 12346, "commented_at": "2026-07-08T10:35:00"}
    ]
  }
  ```
- **接口**:
  - `has_commented(topic_id: int) -> bool` — 检查是否已评
  - `record_comment(topic_id: int)` — 记录已评
  - `load()` / `save()` — 读写文件

### 4.8 `skip_list_manager.py`

黑名单管理（用户指定不评论的帖子）：

- **文件路径**: `skip_list.json`（用户手动创建和维护）
- **数据结构**:
  ```json
  {
    "skip_topic_ids": [12345, 12346, 12347]
  }
  ```
- **接口**:
  - `is_skipped(topic_id: int) -> bool` — 检查是否在黑名单中
  - `load()` / `save()` — 读写文件
- **说明**: 若文件不存在，默认黑名单为空，不报错。用户可随时编辑 `skip_list.json` 增删帖子 ID。

### 4.9 `main.py`（主流程）

执行顺序：

1. 加载配置（`config.py`）
2. 加载已评状态（`state_manager.py`）
3. 加载黑名单（`skip_list_manager.py`）
4. 获取目标分类的最新主题列表（`discourse_client.get_latest_topics()`）
5. 遍历主题列表：
   - 若 `skip_list_manager.is_skipped(topic_id)` 为 True，打印"黑名单跳过"，继续下一个
   - 若 `state_manager.has_commented(topic_id)` 为 True，打印"已评价跳过"，继续下一个
   - 若已达 `MAX_COMMENTS_PER_RUN` 上限，终止循环
   - 获取帖子详情，提取标题 + 正文
   - 调用 `comment_generator.generate(title, body)` 生成评论
   - 调用 `discourse_client.create_reply(topic_id, comment)` 发表评论
   - 若发表成功，调用 `state_manager.record_comment(topic_id)` 记录到本地 JSON
   - 每次评论之间增加 3~5 秒随机延迟，模拟人工操作
6. 打印执行摘要（共处理多少条、成功多少条、跳过多少条、黑名单跳过多少条）

## 六、假设与决策

1. **Cookie 获取方式**：假设用户能够从浏览器开发者工具的 Network 面板中手动复制 Cookie 字符串填入 `.env`，程序不负责自动登录。
2. **Discourse API 可用性**：假设论坛的公开/登录态 API 遵循标准 Discourse 规范（`.json` 后缀获取 JSON 数据）。
3. **目标分类格式**：目标 URL `https://forum.trae.cn/c/38-category/40-category/40/l/new` 中，`38-category/40-category/40` 为分类 slug 和 id，程序将从中解析出 API 调用所需的参数。
4. **DeepSeek 模型名称**：用户指定 `deepseekv4flash`，计划中将映射为 `deepseek-v4-flash`（若 API 实际名称不同，用户可在 `.env` 中调整）。
5. **去重粒度**：以 `topic_id`（主题级别）为去重维度，即一个主题只评论一次，不回复主题下的多个楼层。
6. **安全限速**：默认每次最多评论 5 条，评论间隔 3~5 秒，防止触发论坛反垃圾机制。
7. **黑名单机制**：用户通过维护 `skip_list.json` 文件指定不想评论的帖子 ID，程序在抓取列表后优先检查黑名单，再检查已评状态。文件不存在时默认黑名单为空。

## 七、验证步骤

1. **配置验证**：运行脚本时若 `.env` 缺失或 API Key 为空，立即报错并提示用户。
2. **登录态验证**：首次获取主题列表时，若返回 403/401，提示用户 Cookie 可能失效。
3. **生成验证**：在正式发帖前，可先将生成的评论内容打印到控制台，供用户确认（或提供 `--dry-run` 模式）。
4. **去重验证**：运行两次脚本，第二次应对同一批帖子全部显示"已评价，跳过"。
5. **状态持久化验证**：检查 `state.json` 是否正确写入了已评 topic_id 和时间戳。
