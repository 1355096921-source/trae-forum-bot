# 投票和评论修复计划

## 一、问题分析

测试结果显示两个问题：

### 问题 1：投票 API 404
- 当前调用：`POST /t/22549/vote` → 返回 404 Not Found
- 原因：Discourse Topic Voting 插件的投票端点可能不同

### 问题 2：评论发表 403
- 错误：`请求被禁止 (403)`
- 原因：CSRF Token 获取或使用方式不正确，或 Cookie 失效

---

## 二、排查步骤

### 步骤 1：排查 CSRF Token 获取
- 检查 `/session/csrf.json` 是否能返回有效 Token
- 确认 Token 是否正确添加到请求头

### 步骤 2：排查投票 API 端点
- 通过浏览器开发者工具查看实际的投票请求
- 尝试多种可能的端点：
  - `POST /topics/{id}/vote`
  - `POST /posts/{id}/vote`
  - `POST /t/topic/{id}/vote`

### 步骤 3：检查 Cookie 有效性
- 确认 Cookie 是否过期或格式错误

---

## 三、修复方案

### 方案 1：修复 CSRF Token
- 增加调试日志，显示获取到的 CSRF Token
- 确保所有 POST 请求都携带正确的 Token

### 方案 2：找到正确的投票 API
- 创建调试脚本，逐一尝试不同的投票端点
- 根据结果更新 `discourse_client.py`

---

## 四、文件变更

| 文件 | 操作 | 说明 |
|------|------|------|
| `debug_csrf.py` | 新建 | 调试 CSRF Token 获取 |
| `debug_vote_api.py` | 新建 | 调试投票 API 端点 |
| `discourse_client.py` | 修改 | 修复投票 API 和 CSRF |

---

## 五、风险处理

- Cookie 失效：提示用户重新登录获取
- API 端点不确定：通过调试脚本逐一验证
