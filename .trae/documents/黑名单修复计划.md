# 黑名单功能修复计划

## 问题分析

用户反馈黑名单没有正常跳过帖子（如 21487 和 50091）。

**根因**：`skip_list.json` 文件存在语法错误，第5行 `21487` 后面缺少逗号，导致 JSON 解析失败。

```json
{
  "skip_topic_ids": [
    28826,
    22549,
    21487        // ❌ 缺少逗号
    22548,       // 语法错误！
    ...
  ]
}
```

当 JSON 解析失败时，`SkipListManager` 的 `load()` 方法会捕获异常并将 `skip_ids` 设置为空集合，导致所有黑名单记录丢失。

## 修改方案

### 文件结构

| 文件 | 作用 | 修改方式 |
|------|------|----------|
| `skip_list.json` | 黑名单数据文件 | 修复语法错误 |
| `skip_list_manager.py` | 黑名单管理类 | 增强错误日志 |

### 关键修改步骤

1. **修复 JSON 语法错误**：在第5行 `21487` 后面添加逗号
2. **增强错误提示**：在 `SkipListManager.load()` 中添加解析失败时的警告日志

### 预期效果

修复后，运行 `python main_browser.py` 时：
- 黑名单记录（包括 21487、50091、22549 等）将被正确加载
- 这些帖子会被自动跳过，不再进行投票和评论
- 如果 JSON 文件再次出现语法错误，会有明确的警告日志

## 实施步骤

1. 修复 `skip_list.json` 的语法错误
2. 修改 `skip_list_manager.py` 添加错误日志
3. 测试验证黑名单跳过功能
